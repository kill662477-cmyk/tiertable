**설계 초안 대비 실제로 추가/변경된 로직**:
- 매치타입 배율: 스폰 **×0.5**로 축소 (초안 ×1) — 물량 많은 스폰 위주 반복 대전이 점수를 과하게 좌우해서 축소
- **동일 상대 반복 감쇠**: 같은 날 같은 매치업 3판까지 정상, 4~6판 ×0.5, 7판+ ×0.2 (제로섬 유지) — 특정 상대 "점수 농사" 방지
- **같은 티어 격차 완화**: 같은 티어끼리는 MMR 차이의 체감폭을 절반으로 (SAME_TIER_GAP_DAMPING=0.5)
- **이변 패배 완화**: 하위 티어에게 지면 패자 손실만 ×0.7 (승자 보상은 그대로, 소폭 인플레이션은 소프트리셋이 흡수)
- **신규 수렴 가드**: 신규/복귀자가 아직 배치 전(대국수 30판 미만)일 때, 상대(기존 인원)의 변동을 ×0.4로 축소 — 신규 유입이 기존 인원 점수를 체계적으로 빨아먹는 디플레이션 방지
- **신규도 배치고사 적용**: 초안은 "신규는 유스 600에서 즉시 시작"이었으나, 실측 결과 이게 기존 인원 전체의 체계적 하락을 유발함을 확인 → 신규도 복귀자와 동일하게 10경기 배치고사로 변경
- **배치고사 신뢰도 필터**: 배치 10경기 판정 시 상대도 검증된 인원(시드이거나 본인 판수 10+)일 때만 표본으로 인정 — 미검증 상대끼리의 배치는 의미 없어서 제외
- **3개월 무전적 → 휴면 처리**: 티어표 표시에서 제외(데이터·MMR은 유지). 3개월 공백 후 복귀 시 "복귀자(재측정중)" 뱃지만 달고 MMR은 리셋하지 않음(공백 중 소프트리셋 압축이 자연 감쇠)
- **MMR 바닥 하한선**: 600 고정 — 유스에서 무한정 추락해 영구 승급불가 상태("점수 무덤")가 되는 것 방지
- **승급전/강등 게이트(§4~5) 미구현**: 이번 1차 계산은 순수 Elo + 소프트리셋까지만 — 승급전 5판3선/강등 4조건은 다음 단계 과제로 보류
- **표시 경계선은 관리자 실제 티어표에 회귀 후 수동 미세조정**: 자동 재적합(매 실행 재계산)은 개인 성적과 무관하게 전체가 출렁이는 부작용이 있어 폐기 — 1회 확정한 고정값(`data/calibration.json`)만 사용
- **수동 오버라이드**: 나도현(표시 제외), 보혜(1티어 고정), 다예=얌지금·유복실·상어녀·연둥바둥·삐약삐약·사랑e·으냉이(리네임/로스터이탈 확인되어 개별 보정) — `scripts/lib/aliases.js`

## 1. 기본 공식 — 표준 Elo

```
기대승률  E_A = 1 / (1 + 10^((R_B - R_A) / 400))
변동폭    ΔR_A = K_eff × (S_A - E_A)     // S_A: 승=1, 패=0
갱신      R_A += ΔR_A,  R_B -= ΔR_A     // 제로섬
```

- **승/패만 반영** — 세트 스코어(3:0 vs 3:2) 무시
- **전체 선수 통합 MMR 풀 하나** — 종족(테란/저그/토스) 구분 없음

## 3. K factor

```
K_eff = min( K_base × 매치타입배율, 32 )    // 하드캡 32
K_base = 16
```

**매치타입 배율** — 분류 원천은 eloboard 전적의 **메모란(`memo` 필드)**
- eloboard의 `경기방식` 컬럼(`3/2(2)`, `단판`, `7/4(4)` 등)은 세트 포맷일 뿐 경기 종류 아님
- collect-data.js가 `matchType`(경기방식)·`memo`(메모)를 이미 수집해 Supabase에 저장 중

**메모 키워드 분류 기준 (확정)** — 아래 순서대로 판정, 먼저 걸리는 카테고리 적용

| 순서 | 카테고리 | 키워드 |
|---|---|---|
| 1 | 대회 (×3) | JPL, 큐센, 대학리그, 씨나인배, 라네트배, 숲퍼컵, 스타워즈 |
| 2 | 대학대전 (×2) | "대학대전" 또는 "스타대전" 포함 && **"미니" 미포함** |
| 3 | CK (×1.5) | CK, PL, 미니대전, 끝장전 ("미니" 붙은 대학대전류는 전부 여기) |
| 4 | 스폰 (×1) | 위 어디에도 안 걸리는 나머지 전부 (스폰 명시 포함, 공백 메모 포함) |

- 판정 순서가 중요한 이유: "대학리그"는 "대학"이 들어가도 대회
- 대학대전 vs 미니대학대전 구분 = **"미니" 포함 여부** 하나로 판정
- 기본 카테고리 명칭: 일반 → **"스폰"**으로 통일

| 카테고리 | 배율 | 실효 K |
|---|---|---|
| 스폰 (기본) | ×1 | 16 |
| CK | ×1.5 | 24 |
| 대학대전 | ×2 | 32 |
| 대회 | ×3 | 48 → 캡 32 |

**판수 보정 없음** — 신규/복귀자 포함 전원 동일 K 적용 (초반 판수에 따른 배율 가속 없음)

**하드캡의 의미**: 어떤 조합이든 한 판 최대 변동 ±32.
티어 갭 200점 돌파에 동급 상대 기준 최소 13연승, 이변(한 티어 위 잡기) 연속이라도 최소 9연승 필요.
→ 강등이 보수적이므로 승급도 어렵게 해서 티어 정체(상위 비대화) 방지.

## 4. 승급 (보수적)

**승급전 진입 조건 (모두 충족)**
1. MMR ≥ 상위 티어 기준선 + 30 (턱걸이 방지)
2. 현재 티어에서 15경기 이상 소화
3. 누적 10경기 이상 (표본 부족 상태 승급 금지)

**승급전: 5판 3선승**
- 진입 후 다음 5경기로 진행 (경기 종류 무관 — 미정이면 전 경기 카운트)
- 3승 → 승급 확정 / 3패 → 실패
- 승급전 중 MMR 보정: **승리 ΔR×0.5, 패배 ΔR×1.5**
  → 실력이 확실히 기준선을 넘는 선수만 통과

**실패 페널티**
- 이후 10경기 동안 승급전 재진입 불가 (쿨다운)
- MMR을 기준선 −20으로 조정 (무한 재도전 방지)

## 5. 강등 (매우 보수적 — 4조건 동시 충족)

1. 현재 티어 하한선 미달 상태 **30일 이상 지속** (`demotionBelowSince`)
2. 미달 기간 중 **10경기 이상** 소화 (휴면자는 강등 안 됨)
3. **유예 카운트 소진** (`demotionProtectedCount` — 미달 중 패배 시 차감, 승리 시 일부 회복)
4. 하한선보다 **50점 이상** 깊은 미달

- 강등 확정 시 1티어만 하락, 유예 카운트 리셋
- 목표 보수성: **연간 강등 2명 내외** (과거 1년 시뮬 결과 기준), 승급도 소수 유지
- 세부 수치(30일/10경기/50점/유예 초기값)는 시뮬레이션으로 튜닝

## 6. 소프트 리셋 (6개월마다, 시즌제)

1. **티어 재배치**: 리셋 시점 MMR이 속한 구간으로 티어 확정 (단, 한 번에 최대 1계단만 이동)
2. **MMR 압축**: `newMMR = 새 티어 기준선 + (MMR − 새 티어 기준선) × 0.7`
   - 계수 0.7 = 약한 압축 (시즌 성과 보존 우선)
3. **카운터 초기화**: 승급전 진행/쿨다운, 강등 추적 리셋. **누적 경기수는 유지**

## 7. 신규 / 복귀자

**식별**: eloboard key(userId)로 과거 이력 조회
- 이력 전혀 없음 → 진짜 신규
- 과거 이력 있으나 현 티어표에 없음 → 복귀자

**진짜 신규**
- **유스 티어(600)에서 시작**, 첫 경기부터 정상 계산 (별도 보정 없음)
- 등장 즉시 티어표에 반영 (정기 갱신 안 기다림)

**복귀자 — 배치고사**
- 복귀 후 첫 10경기 미배치 관찰, 상대 티어(MMR)와 승패로 배치:
  `배치 MMR = 상대 평균 MMR + 400 × (승−패) / 경기수`
- 배치 상한: **과거 마지막 티어 초과 배치 불가** (기본값, 재논의 가능)
- 배치 기간 중 상대방(기배치 선수)의 MMR 변동 ×0.5 축소 (점수 풀 오염 방지)
- 측정 완료까지 **"복귀자(측정중)" 뱃지 별도 표기**, 확정 즉시 해당 티어로 게시

## 8. 공개 정책 / 갱신 주기

**공개 범위**
- 일반 공개: **티어만** (MMR 숫자 비공개), 승/패/승률, 신규·복귀자 뱃지
- 관리자 로그인: MMR, 내부 상태 전부 열람

**절대 비공개 (악용 방지)**
- 승급전 진행 여부/전적, 승급 임박, 강등 추적 상태(미달 지속일·유예 카운트·강등 임박), 쿨다운
- 티어 변동은 조용히 진행, **확정 결과만** 게시
- Supabase public API 응답 화이트리스트에서 해당 필드 제외 필수

**갱신 주기**
- MMR: 전적 수집 시마다 내부 계산 (연속)
- 기존 선수 승급/강등 확정 게시: **매달 15일 / 30일** 일괄 반영 (스냅샷 저장)
- 신규·복귀자 배치: **즉시 반영**

## 10. 디스플레이 형식 (티어표 UI)

섹션 순서: **1티어 → 2티어 → … → 8티어 → 유스티어** (총 9개 섹션)
타이틀 명칭: "1티어" ~ "8티어", "유스티어"만 사용. 부제·별명 일절 없음
("베이비 티어", "스타크래프트 입문단계 (언랭)" 등 참고 이미지의 문구는 사용하지 않는다)

```
┌─────────────────────────────────┐
│  〈N티어〉  ← 티어 타이틀만 (부제 없음)
├─────────────────────────────────┤
│  [사진] [사진] [사진] [사진] [사진]   ← 선수 프로필 사진 그리드
│  이름    이름    이름    이름    이름   ← 종족별 색상
└─────────────────────────────────┘
```

- **티어 단위 섹션**: 티어 하나당 한 블록, 그 안에 소속 선수 전원
- **선수 셀**: 사진(정사각형) + 아래 이름
- **이름 색상 = 종족 구분**
  - 테란: **블루**
  - 저그: **보라/핑크**
  - 토스: **노랑**
- **사진 출처**: eloboard 프로필 이미지
  - ⚠ 현재 monstarznew 크롤러(collect-data.js)는 eloboard에서 전적만 파싱하고 사진은 안 가져옴
    (기존 이미지는 players.json에 수동 기재된 SOOP 프로필 URL)
  - → **구현 필요**: eloboard 선수 페이지에서 프로필 사진 추출 → Supabase Storage에 업로드,
    선수 레코드에 이미지 경로 연결
- **뱃지**: 신규 진입자 "New" 표시 (사진 좌상단), 복귀자 "복귀자(측정중)" 표시
- MMR 숫자는 표기하지 않음 (관리자 뷰에서만)
